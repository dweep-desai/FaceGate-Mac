# Security Policy

## Reporting a Problem

**Found a bug, a security issue, or something that doesn't feel right?** Open a public issue on GitHub.

👉 **[Create a new issue](https://github.com/dweep-desai/FaceGate-Mac/issues/new)**

Please include:
- What you found
- Steps to reproduce it
- Your macOS version and Mac model
- Any screenshots or logs (if helpful)

Public issues help everyone see what's being worked on and avoid duplicate reports. No need to email — just open an issue.

## What Happens Next

I'll acknowledge your report within 48 hours and work on a fix as soon as possible.

## Scope

- **Security bugs** — Anything that could leak data, bypass authentication, or compromise the system.
- **Logic bugs** — Features that don't work as expected (schedules, timers, camera, lock/unlock behavior).
- **Crash bugs** — Any reproducible crash.
- **UI/UX issues** — Layout problems, confusing text, accessibility problems.
- **Feature requests** — Not a bug, but still welcome as an issue.

## Supported Versions

Only the latest release on the [releases page](https://github.com/dweep-desai/FaceGate-Mac/releases) receives updates. If you're on an older version, please update first and confirm the issue still exists.

## Hall of Fame

Contributors who report verified issues will be thanked in the release notes. No bounty program at this time — just gratitude and credit.
